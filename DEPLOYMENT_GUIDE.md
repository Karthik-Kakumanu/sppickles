# Deployment Guide - Production Ready

This guide covers deploying the Andhra Kitchen Delights application to production.

```
┌─────────────────────────────┐
```

## Pre-Deployment Checklist

### Frontend
- [ ] Run `npm run build:prod` and verify no errors
- [ ] Check that `dist/` folder contains all assets
- [ ] Update `VITE_API_BASE_URL` to production backend URL
- [ ] Verify all environment variables are set

### Backend
- [ ] Set all required environment variables
- [ ] Database is running and accessible
- [ ] Run migrations if needed
- [ ] Test all API endpoints locally
- [ ] Verify JWT_SECRET is strong and unique

### Security
- [ ] All credentials stored in secure environment variables
- [ ] No hardcoded URLs or API keys
- [ ] CORS origin set correctly
- [ ] HTTPS enforced on all domains

## Frontend Deployment (Vercel)

### Step 1: Create Vercel Project

```bash
# Install Vercel CLI
npm i -g vercel

# Deploy from project root
vercel --prod
```

### Step 2: Configure Environment Variables

In Vercel Dashboard:

1. Go to Settings → Environment Variables
2. Add production variables:

```
VITE_API_BASE_URL=https://api.andhra-kitchen-delights.com
VITE_NODE_ENV=production
```

### Step 3: Update API Base URL

In `src/lib/api.ts`:

```typescript
const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || "http://localhost:5000";
```

The build will automatically use the Vercel environment variables.

### Step 4: Deploy

Push to main branch (if connected to Git):

```bash
git push origin main
# Vercel automatically builds and deploys
```

Or manually deploy:

```bash
npm run build:prod
vercel --prod
```

## Backend Deployment (Railway)

### Step 1: Create Railway Project

1. Go to [Railway.app](https://railway.app)
2. Click "Create Project"
3. Select "Deploy from GitHub" or "Deploy from URL"

### Step 2: Configure Environment Variables

In Railway Dashboard → Variables:

```
NODE_ENV=production
PORT=5000
DB_HOST=<railway-postgres-host>
DB_PORT=5432
DB_NAME=andhra_kitchen_delights
DB_USER=<railway-postgres-user>
DB_PASSWORD=<railway-postgres-password>
JWT_SECRET=<generate-strong-random-key>
CORS_ORIGIN=https://andhra-kitchen-delights.com
FRONTEND_URL=https://andhra-kitchen-delights.com
```

### Step 3: Add PostgreSQL Plugin

1. In Railway project, click "Add"
2. Select "PostgreSQL"
3. Railway automatically provides DB connection variables

### Step 4: Deploy Backend

```bash
# Deploy from repository root
git add .
git commit -m "Production ready backend"
git push origin main
# Railway auto-deploys on push (if configured)
```

Or manually:

```bash
# Build backend (if needed)
# No build step for Node.js, just upload files

# Deploy using Railway CLI
railway up
```

### Step 5: Verify API Endpoint

Test the backend is running:

```bash
curl https://api.andhra-kitchen-delights.com/health
# Expected: { "success": true, "data": { "ok": true } }
```

## Database Setup (PostgreSQL on Railway)

### Step 1: Get Connection Details

From Railway Dashboard:

```
HOST: your-postgres-host.railway.app
PORT: 5432
DATABASE: railway
USER: postgres
PASSWORD: <auto-generated>
```

### Step 2: Run Migrations

Connect to Railway PostgreSQL:

```bash
psql -h <HOST> -p 5432 -U postgres -d railway -f database/schema.sql
```

### Step 3: Verify Tables

```bash
psql -h <HOST> -p 5432 -U postgres -d railway -c "\dt"
# Should show: products, orders, order_items, admin_users
```

### Step 4: Configure Backups

In Railway → Database → Backups:
- Enable daily automated backups
- Retention: 30 days

## Production Troubleshooting

### Frontend won't load

**Check 1:** Vercel deployment logs
```bash
vercel logs
```

**Check 2:** Verify environment variables
- Go to Vercel Settings → Environment Variables
- Confirm `VITE_API_BASE_URL` is set

**Check 3:** Clear browser cache
```
Ctrl+Shift+Delete → Clear all cookies/cache
```

### API calls failing

**Check 1:** CORS is enabled
```bash
curl -i -X OPTIONS https://api.andhra-kitchen-delights.com/products
# Should show: Access-Control-Allow-Origin header
```

**Check 2:** Backend is running
```bash
curl https://api.andhra-kitchen-delights.com/health
```

**Check 3:** Database is accessible
- Check Railway Dashboard
- Verify DB_HOST, DB_USER, DB_PASSWORD

### Database errors

**Check 1:** Connection string is correct
```bash
echo "DB_HOST=$DB_HOST"
echo "DB_NAME=$DB_NAME"
```

**Check 2:** Tables exist
```bash
psql -c "\dt"
```

**Check 3:** Disk space available
```bash
# In Railway dashboard → Resources
```

## Scaling Considerations

### Frontend (Vercel)
- Auto-scales horizontally
- CDN caching enabled by default
- No additional configuration needed

### Backend (Railway)
- Vertical scaling available
- Upgrade node size if CPU/memory bottleneck
- Add database read replicas for high traffic

### Database (PostgreSQL)
- Monitor connection count (max ~100 connections)
- Add connection pooling (PgBouncer) if needed
- Consider read replicas for analytics queries
